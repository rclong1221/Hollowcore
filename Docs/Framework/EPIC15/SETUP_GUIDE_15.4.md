# Epic 15.4 Setup Guide: Camera System Juice

This guide covers the necessary Unity Editor setup for the new camera features: **Smart Occlusion**, **Z-Targeting**, and **FEEL Integration**.

## 1. Smart Occlusion (Anti-Clip)
The camera now automatically detects obstacles to prevent clipping through walls.

### How It Works
- Uses a **SphereCast** from the Player's Head to the Camera position.
- If an obstacle is hit, the camera smoothly zooms in to the player's head.
- **Occlusion Mask**: By default, it hits objects on the following Physics Layers:
    - `Default`
    - `Environment`
    - `Ship`
- **Radius**: Checks for a 0.2 unit radius around the camera line of sight.

### Setup Requirements
1.  Ensure your walls, floors, and large obstacles are on one of the **Occlusion Layers** listed above.
2.  If you have transparent objects (like glass) or trigger volumes that should **NOT** block the camera, ensure they are on a different layer (e.g., `TransparentFX`, `Ignore Raycast`, or a custom layer).

---

## 2. Z-Targeting (Lock-On)
A new "Z-Targeting" system allows players to lock their camera and orientation onto enemies.

### Setup Requirements
1.  **Enemy Layers**: The system scans for targets using Unity Physics. 
    - Enemies **MUST** belong to the `CollisionLayers.Creature` layer configuration. 
    - In the Unity Inspector, ensure your Enemy GameObjects (or ECS Physics Shapes) interact with the `Player` layer and are identifiable as targets.
2.  **Input**: The system uses the **Grab** input action (Key: `G`) to toggle the lock.

### Usage
- **Press 'G'**: Lock onto the nearest enemy within **20 meters**.
- **Press 'G' again**: Release the lock.
- While locked, the camera behaves like a "Strafing" camera, always keeping the target centered.

---

## 3. FEEL Integration (Camera Juice)
We have added support for **Screen Shake** and camera impulses using the **MoreMountains FEEL** asset and **Cinemachine**.

### One-Click Setup
We have laid the groundwork, but you must run the Setup Tool to wire up the components on your Camera Rig.

1.  Open the Unity Editor.
2.  In the top menu, go to **Tools > DIG > Setup Camera Juice**.
    - *Note: If this menu item does not exist, use the "Gameplay Feedback Setup Tool" located in `Assets/Editor/Audio/GameplayFeedbackSetupTool.cs` via the Inspector or by calling its setup method.*
    *(Correction: The tool is currently script-only. Follow the steps below manually if a Menu Item is not available)*

### Manual Setup (If Tool Unavailable)
1.  Locate your **Main Camera** (or Cinemachine Brain) in the scene.
2.  Add the component: `CinemachineImpulseListener`.
    - This component allows the camera to "listen" for shake events generated by explosions, landings, and hits.
3.  Ensure your **Virtual Cameras** (e.g., `Context_ThirdPerson`) have a configured **Noise** profile if you want idle sway, or leave it none for a steady cam.

### Verifying Feedback
1.  Select the `GameplayFeedbackManager` in your scene (under `_Systems`).
2.  Expand the `Land Feedback` slot.
3.  You should see a `MMF_CinemachineImpulse` feedback item.
4.  Test by jumping and landing in Play Mode. The screen should shake slightly on impact.
