# EPIC 15.2 Setup Guide: Input System Modernization

This guide covers the new Input System architecture. **Most setup is now automatic.**

---

## Architecture Overview

```
Hardware (Keyboard/Mouse/Gamepad)
         ↓
DIGInputActions.inputactions (Unity Input Actions Asset)
         ↓
PlayerInputReader.cs (MonoBehaviour, auto-initializes)
    - Implements IGameplayActions
    - Applies gamepad aim acceleration
         ↓
PlayerInputState (Static class)
         ↓
PlayerInputSystem.cs (ECS)
    - Reads exclusively from PlayerInputState
    - No direct hardware access
```

---

## 1. Automatic Initialization (Default)

Both `InputContextManager` and `PlayerInputReader` **auto-create at runtime** using `[RuntimeInitializeOnLoadMethod]`. No manual setup required for basic functionality.

When you enter Play mode, you'll see in Console:
```
[InputContextManager] Auto-initialized
[PlayerInputReader] Auto-initialized
```

> [!NOTE]
> The C# class for Input Actions is auto-generated by Unity when you modify `DIGInputActions.inputactions` and have "Generate C# Class" enabled.

## 2. (Optional) Create Prefabs for Inspector Tuning

If you need to customize gamepad settings via Inspector, create prefabs:

### InputManagers Prefab
1. **Create empty GameObject**: `GameObject > Create Empty` → Rename to `InputManagers`
2. **Add Components**:
   - `InputContextManager` (from `DIG.Core.Input`)
   - `PlayerInputReader` (from `DIG.Core.Input`)
3. **Configure PlayerInputReader**:

| Field | Recommended Value |
|-------|------------------|
| Aim Acceleration Curve | Ease-In-Out (0,0) to (1,1) |
| Stick Deadzone | 0.15 |
| Gamepad Look Sensitivity | 150 |

4. **Save as Prefab**: Drag to `Assets/Prefabs/Core/InputManagers.prefab`
5. **Add to Bootstrap Scene**: The prefab's singleton will take priority over auto-initialization.

## 3. (Optional) Create Feedback Manager Prefab

For FEEL haptic integration:

### Create Base Object
1. `GameObject > Create Empty` → Rename to `FeedbackManager`
2. Add **GameplayFeedbackManager** component

### Create Child MMF_Players
For each feedback type, create a child object with an `MMF_Player` component:

| Child Name | Feedbacks to Add |
|------------|------------------|
| `FireFeedback` | MMF_Haptics, MMF_CameraShake (optional) |
| `DamageFeedback` | MMF_Haptics (strong), MMF_Vignette (optional) |
| `LandFeedback` | MMF_Haptics (short), MMF_CameraShake (optional) |

### Assign References
In `GameplayFeedbackManager`, drag each child MMF_Player to its field.

### Save Prefab
Drag to `Assets/Prefabs/Core/FeedbackManager.prefab`

## 4. Cursor Behavior

Cursor locking is handled via Input Actions:
- **Click in game world** → Locks cursor (via `PlayerInputReader`)
- **Escape key** → Unlocks cursor

The legacy `UnityEngine.Input` cursor code was removed from `PlayerInputSystem` as it's incompatible with Input System only mode.

## 6. Equipment Slot Input System

Equipment switching (keys 1-9) now uses Input Actions instead of legacy keyboard polling.

### Architecture

```
Hardware (Key 1-9 or Gamepad D-pad)
         ↓
DIGInputActions.inputactions (EquipSlot1-9 actions)
         ↓
PlayerInputReader.cs (OnEquipSlot1-9 callbacks)
         ↓
DIGEquipmentProvider.HandleEquipInput(quickSlot)
    - Checks EquipmentSlotDefinition modifier keys (Shift for off-hand, etc)
    - Determines target slot index (0=MainHand, 1=OffHand, etc)
         ↓
PlayerInputState.PendingEquipSlot / PendingEquipQuickSlot
         ↓
PlayerInputSystem → PlayerInput component → Server sync
```

### Adding New Equipment Slots (Beyond 1-9)

If you need more than 9 quick slots (e.g., gadget wheel, consumables):

#### Step 1: Add Input Action
1. Open `Assets/Settings/Input/DIGInputActions.inputactions`
2. In **Gameplay** action map, click **+** to add new action
3. Name it: `EquipSlotX` (e.g., `EquipSlot10`)
4. Set **Action Type**: `Button`
5. Add bindings for keyboard/gamepad
6. **Save** → Unity will regenerate `DIGInputActions.cs`

#### Step 2: Add Callback to PlayerInputReader
Add method to `PlayerInputReader.cs`:
```csharp
public void OnEquipSlot10(InputAction.CallbackContext context)
{
    if (context.performed)
        DIG.Items.DIGEquipmentProvider.Instance?.HandleEquipInput(10);
}
```

#### Step 3: (Optional) Add EquipmentSlotDefinition
If the new slot needs modifier key support (like Shift+10 for off-hand):
1. Create a new `EquipmentSlotDefinition` ScriptableObject
2. Configure `RequiredModifier` and `QuickSlotRange`
3. Add to `DIGEquipmentProvider._slotDefinitions` array

### Testing Equipment Input
1. Press **1-9** in game
2. Console should show:
   - `[DIGEquipmentProvider] HandleEquipInput called: quickSlot=X`
   - `[DIGEquipmentProvider] RequestEquip: Slot=0 QuickSlot=X`
3. Weapon should visually switch

## 7. Adding Other New Input Actions

### 7.1 Edit Input Actions Asset
1. Double-click `DIGInputActions.inputactions`
2. Add new Action to **Gameplay** map
3. Add bindings for Keyboard&Mouse and Gamepad
4. Save and regenerate C# class

### 7.2 Extend PlayerInputReader
A developer must:
1. Implement the new callback in `PlayerInputReader.cs`
2. Add a field to `PlayerInputState.cs` if needed
3. Wire it through `PlayerInputSystem.cs`

## 8. Context Switching (UI Integration)

When opening a menu/inventory, call:
```csharp
InputContextManager.Instance.Push(InputContext.UI);
```

When closing:
```csharp
InputContextManager.Instance.Pop();
```

This automatically:
- Disables Gameplay input
- Enables UI input
- Unlocks cursor

---

## 9. Triggering Haptic Feedback

From any MonoBehaviour:
```csharp
GameplayFeedbackManager.TriggerFire();
GameplayFeedbackManager.TriggerDamage(0.5f); // 0-1 intensity
GameplayFeedbackManager.TriggerLand(1.0f);
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| No movement with gamepad | Check Input Debugger for device detection |
| Character spins wildly | Increase deadzone in PlayerInputReader |
| Haptics not working | Ensure MMF_Player components are assigned |
| Cursor stays locked in menu | Call `Push(InputContext.UI)` when opening |
| Weapon slots not switching | Check Console for `HandleEquipInput` logs |
| `InvalidOperationException` in PlayerInputSystem | Remove any `UnityEngine.Input` calls - use Input Actions only |
